      Start Transmission
      method
      startTransmission
      method startTransmission
      Package: dji.sdk.accessory.speaker
      Description:

      Starts to send a complete audio file to the aircraft, which can be played by the speaker. A transmission session can only be started when the transmission state is in IDLE. If the transmission is started successfully, the state will change to READY_TO_TRANSMIT. Then call paceData to send data to SDK. Call markEOF when all data has been sent to SDK. SDK will maintain a buffer to hold all the data and transmit it to the aircraft. For M2E Speaker, the audio file to transmit should fulfill the following configurations:
      - The speaker only receives raw PCM(Pulse-code modulation) data of an audio file, if the audio file is compressed, PCM extraction is needed before the transmission.
      - The audio channel is mono.
      - The audio sample rate is 44,100 Hz.
      - The PCM (Pulse-code modulation) sample depth is 16 bit per sample.
      - The audio file is stored in the aircraft instead of the speaker.
      开始向飞机发送完整的音频文件，扬声器可以播放该文件。传输会话只能在传输状态为IDLE时启动。如果传输成功启动，则状态将变为READY_to_TTRANSMIT。然后调用spaceData将数据发送到SDK。所有数据发送到SDK后，调用markEOF。SDK将保持一个缓冲区，以保存所有数据并将其传输到飞机。对于M2E扬声器，要传输的音频文件应满足以下配置：-扬声器仅接收音频文件的原始PCM（脉冲编码调制）数据，如果音频文件被压缩，则需要在传输前提取PCM。-音频频道为单声道。-音频采样率为44100 HzPCM（脉冲码调制）采样深度为每个采样16位音频文件存储在飞机中而不是扬声器中。

      Input Parameters:
      @NonNull AudioFileInfo info The information of the file to transmit. It is also used to determine the location the file will be stored in the aircraft.
      要传输的文件的信息。它还用于确定文件将存储在飞机上的位置。
      @Nullable final TransmissionListener listener Inform the progress of the data transfer.
      通知数据传输的进度。
      Pace Data
      method
      paceData
      method paceData
      Package: dji.sdk.accessory.speaker
      Description:

      Sends raw PCM data of the audio file to SDK internal logic. For Mavic 2 Enterprise, the transmission rate is limited to 80kB/s by sdk. Therefore, SDK will cache the paced data to a local file when the rate of pacing data is over 80kB/s. The paced data will be appended to the local file. Call markEOF when all data of the data has been paced to SDK.
      将音频文件的原始PCM数据发送到SDK内部逻辑。对于Mavic 2 Enterprise，sdk将传输速率限制为80kB/s。因此，当起搏数据速率超过80kB/s时，SDK会将起搏数据缓存到本地文件。进度数据将附加到本地文件。当数据的所有数据都已定速到SDK时，调用markEOF。

      Input Parameters:
      @NonNull @Size(MAX_INPUT_SIZE) byte[] data Raw PCM data stream in bytes.
      原始PCM数据流（字节）。
      Return:
      boolean true if data is paced successfully. Otherwise, pacing data is not in the current state or the data size is illegal.
      如果数据进度成功，则为true。否则，起搏数据不处于当前状态或数据大小不合法。
      Mark EOF
      method
      markEOF
      method markEOF
      Package: dji.sdk.accessory.speaker
      Description:

      Marks EOF (end-of-file). It is used to inform SDK that all data of the file has been paced. SDK will continue to transmit data to the aircraft. The state will change to IDLE when all data is transmitted. After calling this method, paceData is invalid until a new transmission session starts. If no data is paced before calling this method, the transmission session will end immediately.
      标记EOF（文件结束）。它用于通知SDK文件中的所有数据都已被调整进度。SDK将继续向飞机传输数据。当传输所有数据时，状态将变为IDLE。调用此方法后，在新的传输会话开始之前，spaceData无效。如果在调用此方法之前没有数据的进度，传输会话将立即结束。

      Return:
      DJIError The occurred error if any.
      发生的错误（如果有）。

      Refresh File List
      method
      refreshFileList
      method refreshFileList
      Package: dji.sdk.accessory.speaker
      Description:

      Refresh the audio file list in the aircraft.
      - If the current state is RESET, SDK will try to fetch the complete file list.
      - If the current state is INCOMPLETE, SDK will only fetch the missing list of files.
      - If the operation is started successfully, the FileListState will become SYNCING.
      - When the file list is refreshed successfully, the FileListState will become UP_TO_DATE.
      刷新飞机中的音频文件列表。-如果当前状态为RESET，SDK将尝试获取完整的文件列表。-如果当前状态为不完整，SDK将只获取丢失的文件列表。-如果操作成功启动，FileListState将变为SYNCING。-成功刷新文件列表后，FileListState将变为UP_TO_DATE。

      Input Parameters:
      final CommonCallbacks.CompletionCallback callback Completion callback to receive the result.
      完成回调以接收结果。

A component will have an index greater than zero when there are multiple components of the same type on the DJI product, and one of the components already has the index 0. For instance, M210 can have two gimbal mounted cameras, and will therefore have two gimbal components with indices 0 and 1, and two camera components with indices 0 and 1. For Matrice 600, there are printed numbers on the battery boxes. The Battery component instance with index 0 corresponds to battery compartment number 1. For Inspire 2 and M200 series, Battery with index 0 corresponds to the battery on the port (left hand) side of the aircraft. For M210, M210 RTK and M300 RTK, Camera and Gimbal with index 0 corresponds to the camera and gimbal on the port (left hand) side of the aircraft. Lidar with index 0 corresponds to the lidar(Zenmuse L1) on the port (left hand) side of the aircraft.
      返回组件索引。索引从零开始。当DJI产品上有多个相同类型的组件，且其中一个组件的索引已为0时，组件的索引将大于零。例如，M210可以具有两个安装在万向节上的相机，因此将具有两个索引为0和1的万向节组件，以及两个索引0和1。对于Matrice 600，电池盒上有印刷数字。索引为0的电池组件实例对应于电池仓编号1。对于Inspire 2和M200系列，索引为0的电池对应于飞机左舷（左侧）的电池。对于M210、M210 RTK和M300 RTK，索引为0的相机和万向节对应于飞机左舷（左侧）的相机和转向架。索引为0的激光雷达对应于飞机左舷（左侧）的激光雷达（Zenmuse L1）。
